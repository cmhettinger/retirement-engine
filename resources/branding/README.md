# Retirement Engine Brand Standards

This document defines the visual identity for the Retirement Engine project. It is the reference for colors, typography, and logo usage across reports, dashboards, websites, documentation, and other project surfaces.

---

# Brand Philosophy

Retirement Engine is designed to feel:

* Clear over ornamental
* Trustworthy over flashy
* Analytical over decorative
* Calm through restraint
* Highly readable and decision-focused

The visual identity should communicate long-term planning, financial clarity, and confidence without adding visual noise.

# Typography

Retirement Engine uses a simple three-font system.

---

## Cinzel

### Purpose

Display font used only for brand identity.

### Usage

* Retirement Engine wordmark
* Major cover titles
* Brand marks
* Very high-level document titles

### Avoid

Do not use Cinzel for:

* Body text
* Paragraphs
* Tables
* UI
* Documentation

Cinzel should remain distinctive through limited use.

---

## Source Sans 3

### Purpose

Primary user interface and document font.

### Usage

* Page titles
* Section headings
* Subheadings
* Body text
* Tables
* Captions
* Metadata
* Navigation
* Dashboards
* Reports
* Presentations
* Website content

### Recommended weights

* Light
* Regular
* Semibold
* Bold

Source Sans 3 should be used for nearly all written communication.

---

## Source Code Pro

### Purpose

Monospaced font for technical content.

### Usage

* Source code
* Terminal output
* SQL
* File paths
* Configuration examples
* Command line snippets

Do not use Source Code Pro for general body copy.

---

# Logo Usage

Approved logo lockups:

* Icon
* Horizontal lockup
* Vertical lockup
* Wordmark

Approved logo colors:

* Black
* White
* Retirement Primary Green
* Retirement Light Grey
* Retirement Dark Grey

Do not:

* Stretch the logo
* Rotate the logo
* Apply effects or shadows
* Change typography
* Substitute colors outside the approved palette

---

# Guiding Principle

Cinzel establishes the Retirement Engine identity.

Source Sans 3 communicates planning, analysis, and reporting.

Source Code Pro communicates implementation.

The overall visual language should remain clean, restrained, and consistent across all Retirement Engine project surfaces.
