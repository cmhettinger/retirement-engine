## Near-Term Priority

The next practical build sequence should focus on making the PDF report excellent with the data already available:

1. Build the multi-section PDF shell and navigation.
2. Add the reusable PDF report data context.
3. Migrate reusable chart generation from `tmp/reporting` into `src/retirement_engine/reports/core/`.
4. Add the current-data chart set.
5. Add the current-data report pages: readiness dashboard, findings, financial snapshot, portfolio, income, spending, cash flow, projections, timeline, methodology, and appendices.
6. Polish the generated PDF with a repeatable render-and-verify workflow.

After the current-data report is strong, continue with Monte Carlo, scenario analysis, optimization, taxes, and then the future workbook spending-model sheets needed for richer scenario comparisons.
